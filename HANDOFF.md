# BookWords handoff

Этот файл нужен, чтобы быстро продолжить работу над проектом на другом компьютере или в новом чате с Codex.

## Проект

- Репозиторий: `big-arch/bookwords-app`
- Публичная ссылка: https://big-arch.github.io/bookwords-app/
- Текущая рабочая версия: `2026-07-11-v22`
- Последний важный коммит: `4f2831e Force fresh PWA updates`

## Что уже сделано

- Приложение BookWords работает как PWA через GitHub Pages.
- Синхронизация переделана с Supabase Auth/email на простой личный код синхронизации.
- Больше не нужен код из письма, magic link, SMTP или переходы из почты.
- Один и тот же личный код вводится на компьютере и телефоне, после этого слова синхронизируются через Supabase.
- Код синхронизации сохраняется на устройстве, поэтому после закрытия и повторного открытия приложение не должно снова просить вход.
- Кнопка `Синхронизировать` вынесена под `Экспорт` / `Импорт` на всю ширину.
- Карточки статистики получили спокойные пастельные цвета:
  - `Знаю` - зеленый;
  - `Новые` - оранжевый;
  - `К повтору` - красный.
- Блок облака сделан проще: пользователь работает с личным кодом, а не с почтой.
- Добавлен более жесткий механизм обновления PWA-кэша, чтобы телефон не держал старую версию.
- Автоперевод английских слов на русский теперь сначала использует Google Translate, затем старый резервный сервис и внутренний мини-словарь.
- При добавлении дубля приложение показывает, в какой книге уже есть слово.
- Во все упражнения добавлена кнопка произношения, а после ответа новое слово появляется только по Enter или кнопке `Следующее слово`.

## Как работает облако

Supabase используется как простое хранилище данных. Таблица:

```sql
create table if not exists bookwords_sync (
  sync_key text primary key,
  updated_at timestamptz not null default now(),
  data jsonb not null default '{}'
);

alter table bookwords_sync enable row level security;

create policy "BookWords can read by sync key"
on bookwords_sync for select
using (true);

create policy "BookWords can insert by sync key"
on bookwords_sync for insert
with check (true);

create policy "BookWords can update by sync key"
on bookwords_sync for update
using (true)
with check (true);
```

Важно: это личное приложение, поэтому защита сейчас держится на случайном личном коде. Для публичного продукта нужно будет усилить безопасность.

## Основные файлы

- `index.html` - разметка, версия приложения, подключение CSS/JS.
- `script.js` - логика приложения, Supabase-синхронизация, PWA-регистрация.
- `styles.css` - внешний вид.
- `sw.js` - service worker и кэш PWA.
- `version.json` - быстрый способ проверить, какая версия опубликована на GitHub Pages.
- `CLOUD_SYNC.md` - заметки по облачной синхронизации.
- `portable/BookWords.html` - переносимая HTML-версия, собирается командой `node build-portable.cjs`.

## Как проверить, что сайт обновился

Открыть:

https://big-arch.github.io/bookwords-app/version.json

Там должна быть версия:

```json
"version": "2026-07-11-v22"
```

Если телефон показывает старую версию:

1. Открыть ссылку в Safari: https://big-arch.github.io/bookwords-app/?fresh=20260709-v20final
2. Обновить страницу.
3. Если старая иконка на домашнем экране все еще открывает старое приложение, удалить старую иконку BookWords и добавить заново через Safari: `Поделиться` -> `На экран Домой`.

## Что делать на другом компьютере

1. Открыть GitHub Desktop.
2. Найти репозиторий `bookwords-app`.
3. Нажать `Fetch origin`, затем `Pull origin`.
4. Убедиться, что последний коммит: `4f2831e Force fresh PWA updates`.
5. Если работа продолжается в Codex, дать ему этот файл `HANDOFF.md` как контекст.
